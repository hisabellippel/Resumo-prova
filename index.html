HTML

O que é uma tag? Uma tag é um marcador de texto. Temos várias tags:
A tag div, a sua função é separar os conteúdos do código HTML em divisões de espaço/conjunto, sem qualquer peso semântico.

✔A tag script, sua função é criar conexões com arquivos.

✔O footer, ele é o rodapé da página, e é uma tag semântica

✔O H1 até H6, definine os títulos e os tamanhos.

✔O p é para fazer parágrafo. O main é onde fica o conteúdo principal da página.

✔Header é o cabeçalho. Body é o corpo do código, tudo que vai ficar visível.

✔A tag label é um texto justificativo do Input, nela você usa a tag form também

✔A tag img, que insere imagens. Section, divide o site em seções.

✔Button, botão, tem duas formas de se usar, type"_" e oneclick

PRINCÍPIOS BÁSICOS DO HTML

Começa com DOC ou !DOCtype. O principal sempre vai ser index.html

Para ligar com o CSS tem que colocar a tag link antes do título do site.

No index.html contêm duas partes quando se primeiro forma:

O head -> que é a parte não visível do código.

E o body -> que é a parte visível do código.

tem 3 peças principais dentro do body:

Header -> cabeçalho

Main ->Principal conteúdo da página

Footer ->que é o rodapé da página.

PRINCÍPIOS BÁSICOS DO CSS

CSS é a abreviação de Cascading Style Sheets (Folha de estilo em casacata).

O que é?Linguagem que define a apresentação visual de um documento HTML

Para que serve?Permite estilizar elementos como cores, fontes, espaçamento, layout, imagens, tabelas, etc.

Você usa ponto para chamar uma classe e hashtag para chamar um ID.

Para ligar com o HTML você usa o link. O body afeta o body.

TAGS DO CSS

✔display: Especifica a saída do elemento e a exibição visual na página.

✔background-color: serve para dar cor ao fundo do site.

✔border: Define todas as propriedades de estilo de borda de uma vez.

✔border-color: Define a cor da borda.

✔width: Define a largura do elemento.

✔font-style: Define o estilo da fonte.

✔text-align: Define o alinhamento horizontal do texto.

GIT

O Git é um projeto de código aberto maduro e com manutenção ativa, ele é um exemplo de DVCS (portanto, Sistema de Controle de Versão Distribuído).

No Git, a cópia de trabalho de todo desenvolvedor do código também é um repositório que pode conter o histórico completo de todas as alterações.

Suporte a vários tipos de fluxos de trabalho de desenvolvimento não lineares, em eficiência em projetos pequenos e grandes

DIFERENÇA ENTRE GIT E GIT HUB E ALGUNS COMANDOS

O Git é uma ferramenta de controle de versões local podendo ser utilizada até em modo offline, o GitHub é uma plataforma online onde você pode hospedar os seus projetos e o Git Bash é um aplicativo para sistemas Windows que utiliza os mesmos comandos de versionamento do sistema utilizado no Mac ou no Linux

Git init: É utilizado para inicializar um repositório Git dentro de um diretório do sistema

Git clone: É utilizado para criar uma cópia de um repositório remoto em um diretório da máquina

Git status: É utilizado para verificar o status de um repositório git

Git add: É utilizado para adicionar arquivos ao pacote de alterações a serem feitas

OUTROS COMANDOS DA FERRAMENTA

Git commit: É utilizado para criar uma nova versão do projeto a partir de um pacote de alterações

Git log: É utilizado para ver o histórico de alterações do projeto.

Git branch: É utilizado para criar novos ramos de desenvolvimento.

Git checkout: É utilizado para navegar entre as versões do projeto.

Git diff: É utilizado para visualizar modificações feitas entre commits.
